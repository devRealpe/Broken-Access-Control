🎯 Escalamiento de Privilegios via API

Estás autenticado como Usuario, el rol con menos privilegios. Sin embargo, el endpoint GET /backend/api/admin/users.php solo verifica que exista una sesión activa (requireAuth()) pero NO verifica que el rol sea admin.

Al hacer clic en el botón de abajo, el frontend llama directamente a ese endpoint administrativo. El servidor responderá con la lista completa de usuarios, sus contraseñas en texto plano y estadísticas del sistema. 


          


          

            

              📖 Explicación de la Vulnerabilidad
            


            

              

                

                  ❌ Código Vulnerable
                

                
// admin/users.php
// ⚠️ Solo verifica autenticación
requireAuth();
// ← cualquier usuario logueado
//   puede ejecutar esto

              


              

                

                  ✅ Código Corregido
                

                
// admin/users.php
// ✅ Verifica rol específico
requireRole('admin');
// ← solo administradores
//   pueden ejecutar esto

                ⚡ Impacto
              

              

                Un usuario normal puede ver
                toda la base de usuarios, incluyendo
                contraseñas en texto plano, correos y roles. Esto permite tomar
                control de cuentas con mayor privilegio (admin, doctor) y
                comprometer completamente el sistema. Clasificado como
                OWASP A01:2021 — Broken Access Control.
